# DeepForge 前端开发文档（Vue 前后端分离）

> 依据《AI 全场景社会仿真与决策推演平台 - 功能文档 (V4.0 旗舰版)》整理，面向 **Python（FastAPI）后端 + Vue 前端** 的分离架构，从 **前端工程** 与 **产品 / UI** 双视角定义界面与实现约束。

## 2. 产品定位与前端承载的核心体验

系统从「数据分析」升级为 **平行世界实验室**：多智能体博弈、非线性对话、A/B 时空对比、图谱与多维可视化。前端需要突出三类体验：

1. **可控感**：创建任务 → 生成沙盒 → 开启推演 → 中途干预 → 报告输出，每一步状态清晰、可回退或取消。
2. **临场感**：SSE 流式呈现对话与事件，避免「整页刷新」；图谱（G6）与桑基、热力等图表与叙事时间线联动。
3. **专业感**：B 端决策工具气质——克制配色、清晰层级、数据密度可控，避免娱乐化或廉价渐变堆砌。

### 2.1 推演产出物模型（结束态交付）

单次推演**收敛或用户主动结束**后，系统在业务上向用户交付 **三件相互关联的产物**（前端需在信息架构与状态机上显式表达，避免用户以为「只有 PDF」）：

| 产物 | 含义 | 前端职责摘要 |
|------|------|----------------|
| **① 预测报告** | 面向决策的可读长文：趋势预测、风险归因、改进建议等；可 **导出 PDF**（由后端生成，前端展示生成进度与下载）。 | 报告页 / Tab、目录锚点、打印样式预览、版本号（同一次 run 可多次重生成需与后端对齐）。 |
| **② 可继续推演的世界** | 当前沙盒在某一时刻的 **完整可续状态**（Agent、关系、环境变量、记忆与规则注入结果等），不是静态截图，而是 **可从该快照再次启动推演** 的「活世界」。 | 展示 **checkpoint / 世界版本**、时间戳与摘要；提供 **从此刻继续推演**、**分叉为新任务**（可选）入口；续跑前可编辑干预预案或变量（若产品允许）。 |
| **③ 结论** | 本次推演的 **精炼判断**（可多条）：一句话结论、可执行建议、关键风险等级等；与报告是 **摘要 vs 全文** 关系，不是重复堆砌。 | 任务详情顶部 **结论条**、或独立「结论」区块；支持复制、钉选到工作台；与报告 Tab 内 **置顶摘要** 二选一或并存（由 PM 定稿）。 |

**体验原则**：三者应在同一任务上下文中 **互相跳转**（从结论跳到报告对应章节、从报告跳转到「当时的世界快照」再点继续），形成「读完 → 认世界 → 再推一轮」的闭环。

## 3. 技术栈建议（与现有规划对齐）

| 层级 | 选型 | 说明 |
|------|------|------|
| 框架 | **Vue 3** + TypeScript + `<script setup>` | 组合式 API 利于流式状态与大型图谱组件生命周期管理 |
| 构建 | Vite | 与 Vue 3 默认搭配 |
| UI 组件 | Element Plus / Arco Design / Naive UI 择一 | 表格、表单、抽屉、步骤条成熟；需与设计 Token 二次封装 |
| 图可视化 | **AntV G6** | 文档中的博弈图谱、动态生长 |
| 图表补充 | ECharts（桑基、热力等） | 与文档中「立场流转」「情绪热力」对应 |
| 实时 | **SSE（EventSource）** 或 `fetch` 流 | 「边推演边展示」 |
| 状态 | Pinia | 任务、沙盒、流式缓冲、图谱快照、**世界 checkpoint、结论与报告元数据** |
| 路由 | Vue Router | 按模式与任务维度拆分 |
| HTTP | Axios / ofetch | 统一拦截、错误与鉴权 |

后端为 **FastAPI + Celery** 时，前端需约定：任务 ID、阶段枚举、SSE 事件类型、干预指令的 JSON 结构。

## 4. 信息架构（IA）与路由建议

```
/                     工作台 / 最近任务
/tasks/new            新建推演（选模式：舆情 / 企业 / 创作 / 法律市监 / 组织行为）
/tasks/:id            任务详情（多 Tab：概览 | 沙盒 | 推演实况 | 图谱 | 对比 | 结论与报告）
/tasks/:id/live       可选：全屏「推演实况」（大屏演示）
/tasks/:id/world      可选：独立「世界快照 / 续推演」页（亦可在概览以卡片呈现）
/settings             个人与组织设置、API Key、模型配置（若产品需要）
```

**新建任务** 对齐文档流程：选择模式 → 上传背景资料 / 规则注入 → 变量设定 → 提交生成沙盒。

**任务详情** 对齐：沙盒 Agent 列表、开启推演、动态干预输入；结束后突出 **结论 + 预测报告 + 可续世界**（见 §2.1）。「结论与报告」Tab 内可采用 **子 Tab**（结论摘要 | 完整报告）或单页分区排版。

**续推演**：从某 `checkpoint_id` 继续时，建议新开路由 `tasks/:id/continue?checkpoint=...` 或在同页打开「续跑配置」侧栏，避免与历史 run 时间线混淆。

## 5. 整体布局（高端、大气的 B 端范式）

### 5.1 壳层（App Shell）

采用 **左侧导航 + 顶栏 + 主内容区** 的经典企业级布局，利于模块多、层级深的产品。

| 区域 | 职责 |
|------|------|
| **顶栏（64px 高）** | Logo / 产品名、全局搜索（任务）、环境标签（dev/staging）、用户菜单、通知 |
| **侧栏（宽度 240px，可折叠至 72px）** | 一级导航：工作台、任务、模板库（可选）、设置 |
| **主内容区** | 页面标题条 + 面包屑 + 操作按钮区 + 卡片化内容 |
| **可选右侧抽屉** | 干预指令、Agent 详情、原始事件日志，不抢占主画布 |

**气质要点**：留白充足、主内容最大宽度建议 **1440px** 内居中或 **fluid + max-width**，避免超宽屏一行文字过长。

### 5.2 推演实况页（高密度信息流）

- **左侧（28%～32%）**：时间线 / 对话流（SSE 追加），支持「仅关键事件」「全文」过滤。
- **中间（40%～48%）**：G6 图谱主画布，全高，工具栏浮于画布上沿（缩放、适应屏幕、布局切换）。
- **右侧（24%～28%）**：选中节点详情、立场统计迷你图、干预输入框（固定底部或抽屉）。

小屏时：**Tab 切换**「流 | 图 | 操作」，图谱优先保证可用性（双指缩放等）。

### 5.3 行业参考 UI 解构（图谱 / GraphRAG 类工作台）

以下对照常见「高端数据产品」界面（**主区大图谱 + 右侧信息卡片 + 右下角实时日志**），与 DeepForge 的「博弈图谱 + 任务阶段 + SSE」高度同构，可作为 **推演实况 / 图谱页** 的备选主布局（与 5.1 壳层叠加使用：进入该页后内容区采用本结构）。

| 区域 | 比例参考 | 做法 |
|------|-----------|------|
| **主画布（左）** | 约 **58%～62%** | G6 全高容器；背景与页面灰底一致或略深一度，突出中间「白底图窗」可选 |
| **侧信息栏（右）** | 约 **38%～42%** | 纵向 **堆叠白卡片**，每张卡片对应流水线一步：如「本体/规则注入 → 沙盒生成 → 推演中 → **结论固化 → 报告生成 → 世界可续**」 |
| **画布内浮层** | 叠在图区右上 | **刷新、全屏、边标签开关** 等轻量工具；不占右侧卡片宽度，保证图最大化 |
| **画布内图例** | 叠在图区左下 | **ENTITY / 关系类型** 小卡：色点 + 文案，与节点配色严格一致（舆情模式可改为立场类型） |
| **系统仪表盘（右下浮窗）** | 固定或可调高度 | **深色终端风**（黑底 + 等宽字体 + 时间戳），承接 SSE 原始事件 / 构建日志；与浅色主界面形成对比，工程师与运营都爱用 |

**右侧单张卡片的叙事结构（可复用到 Vue 组件）：**

1. **角标序号** `01 / 02 / …` + 标题 + 右上 **状态徽章**（如「构建完成」绿框绿字，与参考图一致）。
2. **技术行**：`POST /api/...` 等用 **等宽字体**，与说明正文区分层级。
3. **短说明**：一行到一小段中文描述当前阶段在做什么。
4. **标签组**：实体类型、关系类型、Agent 角色等用 **浅灰底圆角 chip** 横向换行排列，信息密度高但不乱。
5. **关键指标**（可选）：卡片底部用大号数字强调节点数、边数、Agent 数等。

**交互与组件库映射（Vue）：** 卡片 / 徽章 / Switch 可用 Element Plus、Arco、Naive 等现成组件，再通过 **统一圆角（8～12px）+ 轻阴影** 拉齐参考图的「浮起感」；图谱区单独封装 `GraphStage.vue`， props 传入 `showEdgeLabels`、legendItems、toolbarActions。

### 5.4 对比（A/B 时空）

双栏或 **上下分屏 + 同步时间轴 scrubber**，强调「同一时刻两时空状态差异」；差异项用弱高亮边框或左侧色条标识。

## 6. 视觉设计系统（Design Tokens）

以下数值为 **建议默认值**，实现时集中在 CSS 变量或设计 Token 文件（如 `tokens.css`），便于换肤与暗色模式。

### 6.1 色彩

| Token | 用途 | 建议（亮色主题） |
|-------|------|------------------|
| `--color-bg-page` | 页面底色 | `#F4F6F9`～`#F5F7FA`（与常见图谱类产品一致，见 §5.3），避免纯白刺眼 |
| `--color-bg-elevated` | 卡片、侧栏顶栏 | `#FFFFFF` |
| `--color-border-subtle` | 分割线 | `#E8EBF0` |
| `--color-text-primary` | 主文案 | `#1A1D24` |
| `--color-text-secondary` | 次要 | `#5C6370` |
| `--color-accent` | 主操作、链接 | 单一品牌色，如深蓝 `#2563EB` 或青灰蓝 `#0F766E`（二选一贯穿） |
| `--color-accent-secondary` | 图谱内开关、次要强调（可选） | 可与参考 UI 一致用 **紫系**（如 `#7C3AED`），仅用于开关、滑块等，避免与成功/危险语义冲突 |
| `--color-success` / `--color-warning` / `--color-danger` | 状态 | 饱和度适中，避免荧光 |
| **图谱语义色** | G6 边：冲突 / 共识 | 与文档一致：**红系冲突、蓝系共识**，在图例与 Tooltip 中重复说明 |

暗色主题：页面底 `#0B0D12`，卡片 `#141824`，边框 `#2A3142`，主字 `#ECEEF2`，强调色可略提亮。

### 6.2 字体与排版

- **UI 字体**：系统栈 `system-ui, "Segoe UI", Roboto, "PingFang SC", "Microsoft YaHei", sans-serif` 或统一引入 **Inter + 思源黑体**。
- **等宽字体（技术区）**：接口路径、JSON 片段、**SSE / 系统日志**、时间戳统一使用 `ui-monospace, "JetBrains Mono", "Roboto Mono", Consolas, monospace`，字号可比正文小 1px，颜色用 `--color-text-secondary`。
- **字号阶梯**：12（辅助）/ 13（表格）/ 14（正文）/ 16（小标题）/ 20～24（页标题）。
- **行高**：正文 1.5～1.6；表格行高 ≥ 40px，保证可读性。
- **字重**：正文 400～500；标题 600；避免全站大量 700。

### 6.3 圆角（统一「柔和科技」）

| 用途 | 半径 |
|------|------|
| 小型控件（标签、小按钮） | `4px` |
| 输入框、普通按钮、表格内嵌块 | `8px` |
| **卡片、面板、模态框** | `12px` |
| 大型容器 / 图谱外框 | `12px`～`16px` |

原则：**同一层级控件圆角一致**；图谱画布外框可与卡片一致，图内节点圆角略小以区分层次。

### 6.4 矩形与分割

- 卡片默认 **1px 边框** `--color-border-subtle`，hover 时边框略加深或阴影抬升二选一，避免同时强变化。
- 列表行用 **底部分割线** 或 **斑马纹（极浅）** 二选一，不要混用。

### 6.5 阴影（Elevation）

克制使用，体现「层级」而非「装饰」：

| 层级 | 示例 | CSS 参考 |
|------|------|----------|
| 0 层（平铺） | 页面上的次级块 | 无边框或仅边框 |
| 1 层（卡片） | 工作台任务卡片 | `0 1px 2px rgba(15,23,42,0.06)` |
| 2 层（下拉、弹出） | Select、Popover | `0 4px 16px rgba(15,23,42,0.08)` |
| 3 层（抽屉、模态） | 干预确认、详情抽屉 | `0 12px 40px rgba(15,23,42,0.12)` |

**禁止**：大范围强扩散 + 高不透明度，易显廉价。

### 6.6 间距与栅格

- 基准 **8px 栅格**：4 / 8 / 12 / 16 / 24 / 32 / 48。
- 卡片内边距：**20px～24px**；区块间距：**24px～32px**。
- 表单标签与控件纵向间距 **8px～12px**。

### 6.7 动效

- 时长：**150ms～220ms**（进入），**120ms～180ms**（离开）。
- 缓动：`cubic-bezier(0.4, 0, 0.2, 1)`。
- SSE 新消息入场：**轻微 fade-in 或高度展开**，避免闪烁跳动；图谱节点生长用 G6 内置动画，时长与全局动效协调。

## 7. 关键界面与组件清单

| 模块 | 主要组件 | 前端要点 |
|------|-----------|----------|
| 新建任务 | 步骤条、模式选择卡片、文件上传、动态表单（变量） | 大文件分片上传若后端支持；校验与草稿本地缓存 |
| 沙盒概览 | Agent 表格 / 卡片墙、权重展示（1:9:90） | 虚拟滚动（量大时） |
| 推演实况 | 虚拟列表对话流、SSE 重连、暂停/继续 | `EventSource` 断线指数退避；防抖合并渲染 |
| 图谱 | G6 封装组件、图例、缩放工具栏 | 与 Pinia 中「图快照」同步；销毁时 `graph.destroy()` |
| 多维可视化 | ECharts 桑基、热力地图层 | 与任务维度配置联动 |
| 干预 | 指令输入、确认、时间戳绑定 | 与后端「在 T 时刻注入」语义一致 |
| **结论** | 可折叠列表、优先级标签、复制/导出片段 | 与报告区分：短句 + 可执行项；可引用 checkpoint |
| **预测报告** | Markdown/HTML 渲染、目录、PDF 链接 | 生成状态机：排队 / 生成中 / 成功 / 失败；多版本需列表 |
| **可续世界** | 快照时间线、checkpoint 卡片、「从此继续」主按钮 | 展示世界版本 ID、关联 run；续跑前确认资源与计费（若产品需要） |

## 8. 与后端协作约定（建议写入 OpenAPI）

- **REST**：任务 CRUD、沙盒生成、启动/停止推演、干预提交、**结论查询**、**报告生成与下载状态**、**世界快照列表与详情**、**从 checkpoint 继续推演 / 分叉新任务**（路径与动词由后端定稿，前端需类型化 DTO）。
- **SSE**：统一路径如 `GET /api/tasks/{id}/stream`，事件类型枚举：`agent.message` | `graph.update` | `metrics.tick` | `phase.change` | **`conclusion.ready`** | **`report.ready`** | **`world.checkpoint`** | `error`。
- **鉴权**：Bearer Token；SSE 需支持 query 或 cookie 方案（视部署而定）。
- **错误体**：统一 `{ code, message, details }`，前端全局拦截 + 页面级兜底。

## 10. 非功能需求

## 11. 验收与迭代节奏（产品视角）

1. **MVP**：新建任务 → 列表 → 详情 → SSE 对话流 + 基础 G6 + **结论展示 + 报告下载 + 世界快照列表 + 从最新快照继续推演（单一路径）**。
2. **V2**：A/B 对比视图、桑基/热力、干预与「关键一句话」高亮联动。
3. **V3**：大屏模式、模板库、组织级权限与审计日志。

---

## 12. 结语

前端不仅是「壳子」，而是 **推演可信度与决策安全感** 的载体：布局稳定、视觉克制、实时反馈可靠、图谱与叙事一致；推演结束后，用户应能 **带走结论、带走报告、带走一条可再推演的世界线**（见 §2.1）。本文档可与接口文档、Figma（若后续补充）并行维护，版本号建议与后端 **V4.0** 对齐迭代。
